순수 HTML로만 구성된 웹페이지

HTML만 사용하여 제작된 웹페이지입니다. 브라우저 기본 스타일로만 표현됩니다.

배경 (Background)

HTML의 역할

HTML(HyperText Markup Language)은 웹페이지의 뼈대와 의미를 정의하는(구조화) 마크업 언어입니다.

  • 문서의 뼈대 만들기(구조화) - 제목, 단락, 목록 등
  • 의미를 담은 태그(시맨틱 요소)를 통한 의미 부여
  • 링크를 통한 연결된 문서(하이퍼텍스트) 구현
  • 접근성을 위한 기본 골격 제공

효과 (Effects)

분명한 뼈대(구조적 명확성)

제목, 본문, 목록 등이 순서대로 정리되어(계층적 구조화) 정보의 흐름이 논리적입니다.

접근성

스크린 리더와 같은 보조 기술이 문서 뼈대(구조)를 쉽게 파악할 수 있습니다.

호환성

모든 브라우저에서 기본적으로 지원되며, 인터넷 환경에 관계없이 빠르게 로딩됩니다.

결과 (Results)

"내용이 형식보다 우선이다. HTML만으로도 충분한 정보 전달이 가능하다."

HTML 단계의 특징
항목 상태 설명
시각적 표현 기본 브라우저 기본 스타일만 적용
상호작용 없음 링크 외에 동적 기능 없음
로딩 속도 매우 빠름 추가 리소스 없이 텍스트만 전송
접근성 우수 시맨틱 구조로 보조 기술 호환

한계 (Limitations)

  1. 단조로운 시각적 표현

    브라우저 기본 스타일만 사용하므로 시각적 매력이 부족합니다.

  2. 사용자 경험 제한

    상호작용이나 동적 기능이 없어 현대적인 웹 경험을 제공하기 어렵습니다.

  3. 브랜딩 불가

    고유한 디자인이나 브랜드 아이덴티티를 표현할 수 없습니다.

  4. 반응형 디자인 부재

    다양한 디바이스에 최적화된 레이아웃을 제공하기 어렵습니다.
